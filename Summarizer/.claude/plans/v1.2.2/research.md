## 목표사항

- 직원 이름을 사용자로부터 입력 받을 수 있는 입력 컨트롤 UI를 제공
  - 앱 윈도우 UI의 좌측 상단에 제공한다.
  - `AppSettings.json`의 `staffName`과 연동된다.

- 여러 개의 대화를 제공하여 변환하는 기능
  - 하나의 텍스트 파일 안에, 일정한 문법 형식을 갖추어 여러 벌의 대화를 구분하여 작성한다.
  - 각 대화 한 벌은, 기존의 텍스트 변환 기능을 이용하여 변환한다.
  - 변환한 텍스트들은 입력 받은 텍스트 파일과 동일한 위치에, 새로운 텍스트 파일 하나에 모아서 내보낸다.
  
- 여러 대화를 불러오기 위한 텍스트의 명세
  - 텍스트 파일 형식 : 기본은 .txt를 사용하나, 특별히 확장자를 구분하지 않고, 불러온 텍스트는 모두 일반적인 plaintext로 취급한다.
  - 텍스트 인코딩 : (문제를 단순하게 하기 위해)모두 UTF-8(No BOM)로 취급한다.
  - 각 대화를 구분하는 문법 기호 : bracket([]) 사용
  - [] 안에 있는 텍스트들은 대화 한 벌로 간주한다.
  ```
    [
        대화1의 1번째 줄
        대화1의 2번째 줄
        대화1의 3번째 줄
        ...
    ]
    [
        대화2의 1번째 줄
        대화2의 2번째 줄
        ...
    ]
    [
        ...
    ]
  ```
  - [] 안에 들어가지 않은 문자열들은 대화로 취급하지 않고 변환 대상에서 무시한다.
  - 각 대화들은 변환한 결과 텍스트들은 대화 하나당 한줄씩 내보내기 할 텍스트에 출력한다.
  - 내보낼 텍스트의 이름은, 입력 받은 텍스트 파일 이름에 `-converted`를 붙이고, 동일한 확장자를 사용하여 만든다.
  - 내보낼 텍스트는 입력 받은 텍스트 파일과 동일한 인코딩을 사용하고, 동일한 디렉토리 위치에 저장한다.

- 위 명세의 텍스트 파일을 이용해 여러 대화를 불러오고 -> 변환하고 -> 내보내는 과정을 명령행 인터페이스로도 수행할 수 있게 한다.
  - 기존 앱에 매개변수를 지정하여 실행하는 방법을 지원하거나, (가장 선호)
  - 이를 수행할 수 있는 콘솔 앱을 별도로 만든다. (불가피한 경우의 차선책)
