## Summarizer v1.2.1 계획

---

## 기본사항

- 사용 버전 : v1.2.4
- 이전 버전 : v1.2.3
- 이전 버전에서 작업했던 내용들을 이어서, 여기에서 작업해라.

---

## 목표사항

- 특정한 형식을 가진 알림 메시지 텍스트를 입력 받아, 이를 원하는 형식으로 변환하는 기능을 추가한다.
  - 원본 텍스트 : `강남언니` 서비스로부터 받은 모객 관련 알림 메시지
  - 변환한 텍스트 : 기존의 KakaoTalk Business 메시지의 변환 결과와 마찬가지로, 각 항목을 핵심 텍스트들만 추려내서 '/' 기호로 구분한 텍스트로 만들어낸다. 
  
- 변환할 텍스트를 입력받는 창 및 결과 텍스트를 표시하는 창은 기존 앱의 입력창 GUI를 그대로 사용한다. 
  - 프로그램 내부적으로 처리 분기만 달라짐.
  - 즉, 사용자는 새 기능을 위해 별개의 UI를 사용해야 할 필요가 없다.

---

## 설계 지침

- 추가하고자 하는 모객 서비스(`강남언니`)의 고객 예약 관련 메시지들은 다음과 같은 종류로 분류하여 처리
  - 신규 예약
  - 예약 변경
  - 예약 취소

- 주어진 텍스트의 가장 첫번째 줄의 텍스트 내용을 통해, 어떤 분류의 모객 메시지인지 분류한다.
  - 모객 메시지의 세부 내용에 따라, 변환 처리할 정책을 분기한다.
  - 첫줄의 텍스트가 시간 표시라면, KakaoTalk Business 메시지로 분류하여 기존의 구현대로 처리할 수 있다.

- 모객 서비스는 현재는 `강남언니` 만 대상이지만, 향후 다른 서비스의 모객 메시지도 유사하게 처리할 가능성을 염두하여 구현한다.

## 모객 메시지 텍스트 변환의 세부사항

### 일러두기

- $ 기호 안의 내용은 변수 텍스트임.
- {ex. } 기호 안의 내용은 예시 텍스트임.
- | 기호는 열거 항목 중에서 1개가 나타남을 의미
- ... 기호는 바로 직전 형식의 텍스트가 가변 개수로 나타날 수 있음을 의미
- 그 외에는 상수 텍스트임.

### 신규 고객 예약

- 원본 텍스트 형식
```
고객이 예약을 신청했어요.
예약 일시를 확정해 주세요.

[고객명] : $고객이름$
[연락처] : $휴대전화번호$ {ex. 010-0000-0000}
[예약 항목] : (병원) | (의사) $원장 성명$ | (이벤트) 눈성형 중점 $진료항목1 진료항목2 ...$
[희망 일시] : 
1. $날짜와 요일-1$ $시간-1$ {ex. 2025. 5. 12 (월) 오전 10:00}
2. $날짜와 요일-2$ $시간-2$
3. $날짜와 요일-3$ $시간-3$
...
```

- 변환 결과 텍스트 형식
```
강남언니 / $고객이름$ / $휴대전화번호$ / $원장 성명$ | $진료항목1 진료항목2 ...$
1. $날짜와 요일-1$ $시간-1$ / 2. $날짜와 요일-2$ $시간-2$ / 3. $날짜와 요일-3$ $시간-3$ ...
```

- 지침
  - `[예약 항목]` 부분의 `(병원)` 또는 `(이벤트) 눈성형 중점` 텍스트는 변환 결과에서 생략할 것
  - `[희망 일시]`에 해당하는 날짜와 시간값들은 결과 텍스트의 2번째 줄로 기록해야 한다.

### 고객 예약 변경

- 형식
```
고객이 일정 변경을 요청했어요.
변경 희망 일시를 확정해 주세요.

[고객명] : $고객이름$
[연락처] : $휴대전화번호(010-0000-0000)$
[예약 항목] : (병원) | (의사) $원장 성명$ | (이벤트) 눈성형 중점 $진료항목1 진료항목2 ...$
[변경 희망 일시] : 
1. $날짜와 요일-1$ $시간-1$ {ex. 2025. 5. 12 (월) 오전 10:00}
2. $날짜와 요일-2$ $시간-2$
3. $날짜와 요일-3$ $시간-3$
...
```

- 변환 결과 텍스트 형식
```
강남언니 / $고객이름$ / $휴대전화번호$ / 예약 변경
1. $날짜와 요일-1$ $시간-1$ / 2. $날짜와 요일-2$ $시간-2$ / 3. $날짜와 요일-3$ $시간-3$ ...
```

- 지침
  - `[변경 희망 일시]`에 해당하는 날짜와 시간값들은 결과 텍스트의 2번째 줄로 기록해야 한다.
  - 예약 변경인 경우, `[예약 항목]`의 내용에 해당하는 텍스트는 기록하지 않음.
  - 그 대신, `예약 변경` 텍스트만 추가한다.

### 고객 예약 취소

- 원본 텍스트 형식
```
고객이 예약을 취소했어요.
취소 내역을 확인해 주세요.

[고객명] : $고객이름$
[연락처] : $휴대전화번호(010-0000-0000)$
[예약 항목] : (병원) | (의사) $원장 성명$ | (이벤트) 눈성형 중점 $진료항목1 진료항목2 ...$
[취소된 내원 일시] : 확정 전 취소
...
```

- 변환 결과 텍스트 형식
```
강남언니 / $고객이름$ / $휴대전화번호$ / 예약 취소
```

- 지침
  - 예약 취소인 경우, `[예약 항목]`의 내용에 해당하는 텍스트는 기록하지 않음.
  - 그 대신, `예약 취소` 텍스트만 추가한다.
